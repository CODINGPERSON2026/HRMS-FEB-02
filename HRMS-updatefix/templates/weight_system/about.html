Project Overview
The Weight Management System helps track and manage weight data of soldiers efficiently. It calculates ideal weight ranges, identifies fit/unfit personnel, and provides detailed analytics by rank, company, and category. The system enhances health monitoring and reporting.
User Guide
Instructions for using the system include adding users, viewing dashboards, checking fit/unfit statistics, and generating reports. Users can filter data by company or status type.
Technical Documentation
Built with Flask (Python) and MySQL. Frontend uses Bootstrap and Chart.js for responsive UI and dynamic graphs. Backend provides REST APIs for data management and computations.
How Fit/Unfit is Calculated
The system calculates a soldier's fitness status based on their actual weight compared to an ideal weight determined by their age and height:
  • Step 1: Round the height to the nearest even number.
  • Step 2: Look up the ideal weight from predefined tables based on age and height.
  • Step 3: Calculate the acceptable weight range:
    • The weight range can be more than 10% of the actual weight
    • The weight range can be less than 10% of the actual weight
  • Step 4: Compare actual weight to the range:
    • If actual weight is within the range → Status = Fit
    • If actual weight is below or above the range → Status = UnFit
  • Step 5: Calculate weight deviation in kg and percentage for unfit personnel.
This ensures accurate monitoring and reporting of soldier fitness for each company and rank.
Features
  • Calculate ideal weight based on age and height
  • Track actual weight and determine fit/unfit status
  • View safe and category counts
  • Dynamic dashboards with graphs and summaries
  • Detailed user profiles and reporting
  • Real-time updates using AJAX
